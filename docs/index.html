Product Overview

The software designed and deployed by IE BestBank is a banking system allowing users to register and create accounts seamlessly and securely, while also allowing the ability to deposit funds as well as tranfer and receive from multiple accounts. The software is designed to be user-friendly and intuitive, allowing users to easily navigate through the system.

Relevant GitHub Repositories

Below are the relevant GitHub repositories for the IE BestBank project:

  • GitHub Frontend Repository: Contains the frontend code and resources for the IE Bank application, built with Vue.js.
  • GitHub Backend Repository: Contains the backend code and resources for the IE Bank application, built with python.
  • GitHub Infrastructure Repository: Contains the infrastructure code and resources for the IE Bank application, built with Bicep and IaC code.

Functional and Non-Functional Requirements

Below are the new functional and non-functional requirements for the IE BestBank application:

Requirement ID Type Description
FR 1 Functional The application must provide a default administrator account (username and password). The admin account must have access to the user management portal upon successful login.
FR 2 Functional Once logged into the bank users management portal, a bank administrator can list, create, update, and delete bank users and their passwords.
FR 3 Functional New bank users can register on IE Bank using a registration form that includes username, password, and password confirmation. Upon registration, a new account with a randomly generated account number is automatically provided.
FR 4 Functional Bank users can log in to the web application using their username and password. After logging in, they can view only their own bank accounts and transactions.
FR 5 Functional Bank users can transfer money to other existing accounts within the bank through the account management portal by entering the recipientâ€™s account number and the transfer amount. The transfer amount cannot exceed the available balance in the user's account.
NFR 1 Non-Functional The web application must implement a basic user/admin authentication system requiring users to enter their username and password to log in. Advanced authentication methods such as biometric, token, or OAuth should not be used. User credentials must be securely encrypted (hashed) and stored in the database.
NFR 2 Non-Functional The web application should feature a simple frontend user interface, focusing less on aesthetic aspects like colors, fonts, or animations. It is not required to ensure frontend compatibility and responsiveness across different browsers and devices.
